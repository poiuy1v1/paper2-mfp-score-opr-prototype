# Repository DOI insertion map

Replace `[INSERT VERSIONED ZENODO DOI FOR FINAL RELEASE]` after the final GitHub/Zenodo release.

## Required locations

- manuscript Data and code availability section
- `submission_materials/Data_Code_Availability_Submission_Draft.md`
- `submission_materials/Cover_Letter_Draft_DigitalDiscovery.md`
- `SupplementarySoftware1/CITATION.cff`
- `SupplementarySoftware1/README.md`
- `README.md`
- RSC submission portal metadata

## Recommended DOI form

Use the version-specific DOI for the submitted release, not only the concept DOI, unless the journal specifically asks for a concept DOI.
