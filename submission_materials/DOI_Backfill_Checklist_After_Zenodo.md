# DOI backfill checklist after Zenodo release

After GitHub/Zenodo generates a versioned DOI, replace `[INSERT VERSIONED ZENODO DOI FOR FINAL RELEASE]` in:

- manuscript Data/code availability section
- `submission_materials/Data_Code_Availability_Submission_Draft.md`
- `submission_materials/Final_Data_Code_Availability_With_Placeholders.md`
- `submission_materials/Cover_Letter_Draft_DigitalDiscovery.md`
- `SupplementarySoftware1/CITATION.cff`
- `SupplementarySoftware1/README.md`
- root `README.md`
- RSC submission portal metadata

Use the version-specific DOI for the exact submitted release. Keep the concept DOI separately if Zenodo provides both.
